🔐 FLUXO DE AUTENTICAÇÃO
────────────────────────────────────────────────────────────────────────────────

GET /entra/login
  ├─ Gera CSRF token (state parameter)
  ├─ Constrói URL de autenticação
  └─ Redireciona para: https://login.microsoftonline.com/{TENANT_ID}

Usuário faz login no Entra ID
  └─ Fornece credenciais corporativas

Entra ID redireciona para /entra/callback
  ├─ URL: http://localhost:5000/entra-callback?code=AUTH_CODE&state=CSRF_TOKEN
  └─ Backend processa...

GET /entra/callback
  ├─ [1] Valida CSRF token ✓
  ├─ [2] Captura código de autorização ✓
  ├─ [3] Troca código por tokens (usa CLIENT_SECRET) ✓
  ├─ [4] Busca dados do usuário (Microsoft Graph API) ✓
  ├─ [5] Valida email no banco de dados ✓
  ├─ [6] Salva dados na sessão Flask ✓
  └─ [7] Redireciona para página principal

GET /entra/logout
  ├─ Limpa sessão Flask
  ├─ Faz logout no Entra ID
  └─ Redireciona para home


📌 VARIÁVEIS DE AMBIENTE NECESSÁRIAS
────────────────────────────────────────────────────────────────────────────────

No arquivo .env, adicione:

    ENTRA_CLIENT_ID=<seu-client-id>
    ENTRA_CLIENT_SECRET=<seu-client-secret>
    ENTRA_TENANT_ID=<seu-tenant-id>
    ENTRA_REDIRECT_PATH=/entra-callback
